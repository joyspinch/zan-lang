解决的痛点The pain we solve

跨平台 UI，本不该这么重Cross-platform UI shouldn't be this heavy

现有方案要么臃肿、要么门槛高、要么依赖庞大运行时。Zan 想让它重新变得简单、轻快、漂亮。 Existing stacks are bloated, hard, or drag a huge runtime behind them. Zan makes it simple, fast, and beautiful again.

告别 Electron 臃肿No more Electron bloat

一个空壳应用动辄上百 MB、几百 MB 内存。Zan 单文件 <20MB、内存 <100MB，启动即开。 An empty Electron shell is hundreds of MB and RAM. Zan ships a <20MB binary, <100MB RAM, instant launch.

无需 .NET 运行时No .NET runtime

C# 做跨平台 UI（MAUI/Avalonia）要背 .NET 运行时、忍 GC 停顿、配环境。Zan 直接 AOT 出原生单文件，ARC 无 GC。 C# cross-platform UI (MAUI/Avalonia) needs the .NET runtime, GC pauses, and setup. Zan is AOT native with ARC — no GC.

比 C++/Qt 好上手Easier than C++/Qt

Qt/C++ 语言复杂、许可繁琐。Zan 用 C#/Java 风格语法，会 C# 就会写，MIT 完全免费。 Qt/C++ is a hard language with licensing friction. Zan uses C#/Java-style syntax — if you know C#, you know Zan. MIT, fully free.

告别环境配置地狱No setup hell

不用装一堆 SDK、配 PATH、挑版本。下载一个 IDE，新建项目即可写、运行、发布。 No SDK zoo, no PATH juggling, no version roulette. Download one IDE, create a project, write, run, ship.

告别分发噩梦No packaging nightmare

不用带一堆 dll/依赖/安装器。产物是单个原生可执行文件，拷贝过去就能跑。 No dll bundles, no runtimes, no installers. Output is a single native executable — copy and run.

技能零成本迁移Bring your C# skills

using Zan.Game;

class Pong : Game {
    Vec2 ball = new Vec2(480, 300);
    Vec2 vel  = new Vec2(4, 3);

    override void Update(float dt) {
        ball += vel;
        if (ball.Y < 0 || ball.Y > 600) vel.Y = -vel.Y;
    }

    override void Draw(Canvas c) {
        c.Clear(Color.Black);
        c.FillCircle(ball, 8, Color.White);
    }
}
