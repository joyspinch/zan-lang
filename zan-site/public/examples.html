server-mvc — 注解式 Web 框架— annotated web framework

一个封装完整、开发友好的 MVC 框架：路由由控制器目录结构自动推导，请求方法与权限用属性声明，统一安全取参。 A fully-encapsulated, developer-friendly MVC framework: routes derive from the controller directory layout, method/permission declared via attributes, unified safe input filtering.

  • 属性：[Get]/[Post]/[Put]/[Delete]、[Route("/x")]、[Login]、[Auth]、[Rank(n)]、[Lock("user"|"global")]、[Limit(n)]、[Menu]，也支持合并写法 [Api(post, route="/x", auth, rank=9, menu)] Attributes: [Get]/[Post]/[Put]/[Delete], [Route], [Login], [Auth], [Rank(n)], [Lock], [Limit], [Menu], plus a merged form [Api(post, route="/x", auth, rank=9, menu)]
  • 统一取参：ctx.In / InText / InInt / InRaw / HasIn（trim、去控制字符、HTML 转义） Unified input: ctx.In / InText / InInt / InRaw / HasIn (trim, strip control chars, HTML-escape)
  • 构建期生成器 RouteGen 扫描属性 → 生成 Routes.gen.zan，main.zan 一行 Routes.Map(app) 注册 Build-time RouteGen scans attributes → generates Routes.gen.zan; one line Routes.Map(app) registers all
controller/ApiController.zan
[Api(post, route="/api/gather", auth, login, lock="user")]
static void Gather(HttpContext ctx) {
    var id = ctx.InInt("id");
    ctx.Json("{\"ok\":true}");
}

怎么跑：新建 server-mvc 项目 → F5 → 浏览器打开输出里的监听地址。 Run it: create a server-mvc project → F5 → open the printed listening address.

server-iot — EMQX 式 MQTT 物联网服务端— EMQX-style MQTT IoT server

内置一个 MQTT 3.1.1 broker + Web 管理控制台。A built-in MQTT 3.1.1 broker plus a web management console.

  • Broker：CONNECT / PUBLISH(QoS 0/1) / SUBSCRIBE / UNSUBSCRIBE / PING / DISCONNECT，+ / # 通配主题路由，二进制帧编解码 Broker: CONNECT / PUBLISH(QoS 0/1) / SUBSCRIBE / UNSUBSCRIBE / PING / DISCONNECT, + / # wildcard routing, binary frame codec
  • 管理：客户端/设备管理、订阅管理、主题管理、发布接口（踢下线/注入消息）、性能检测（在线数/吞吐/字节/运行时长） Management: clients/devices, subscriptions, topics, publish endpoints (kick/inject), performance metrics (connections/throughput/bytes/uptime)
  • Web 控制台每 2 秒轮询 /iot/* 展示实时数据；复用 server-mvc 注解式路由 The console polls /iot/* every 2s for live data; reuses the server-mvc annotated routing

已通过 14/14 端到端测试：真实 MQTT 客户端订阅 sensors/# → 另一客户端发布 → 订阅方收到；HTTP 注入发布 delivered=1；未授权发布 401。 Passes 14/14 end-to-end tests: a real client subscribes to sensors/#, another publishes, the subscriber receives; HTTP-injected publish delivered=1; unauthorized publish 401.

server-http

最精简的 HTTP 服务：直接用 HttpServer 处理请求、返回 HttpResponse。想完全掌控、不要框架约定时用它。 The barest HTTP server: handle requests with HttpServer and return HttpResponse. Use it when you want full control without framework conventions.

server-tcp

基于 TcpListener / TcpClient 的原始 TCP 服务，演示连接管理与字节协议——自定义二进制协议的起点。 A raw TCP server on TcpListener / TcpClient, showing connection management and byte protocols — a starting point for custom binary protocols.

server-ws-gateway

WebSocket 网关：接受 WS 连接、广播/转发消息，适合实时推送、聊天、在线协作。 A WebSocket gateway: accept WS connections, broadcast/forward messages — great for realtime push, chat and collaboration.

贪吃蛇 / 打砖块Snake / Breakout

examples/game/snake 与 examples/game/breakout：入门级街机游戏，演示固定步长游戏循环、输入处理、碰撞检测与逐帧绘制——最适合第一次写游戏。 examples/game/snake and examples/game/breakout: beginner arcade games showing the fixed-step loop, input handling, collision detection and per-frame drawing — perfect for your first game.

五子棋 / 象棋 / 围棋Gomoku / Xiangqi / Weiqi

gomoku、xiangqi、weiqi：基于 Game.Board 的棋类，演示网格、合法走子校验、回合与胜负判定。 gomoku, xiangqi, weiqi: board games on Game.Board, showing grids, legal-move validation, turns and win detection.

斗地主Dou Dizhu

examples/game/ddz：基于 Game.Cards 的卡牌对战，演示牌型、区域、发牌与出牌规则。 examples/game/ddz: a card battle on Game.Cards, showing hand types, zones, dealing and play rules.

RTS / Mir2

examples/game/rts 演示即时战略核心（单位、地图、指令），并配套 Game.Rts.Formats 解析经典资源格式（INI / PAL / SHP / TMP / CSF）。examples/game/mir2 展示更完整的地图/角色/联网型游戏，对应 Game.Zgm 运行时。 examples/game/rts shows RTS core (units, maps, commands) with Game.Rts.Formats parsing classic assets (INI / PAL / SHP / TMP / CSF). examples/game/mir2 shows a fuller map/role/networked game on the Game.Zgm runtime.
