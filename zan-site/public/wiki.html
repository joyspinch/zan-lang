开箱即用Out of the box

下载、解压、双击运行——编译器与标准库已内置，无需安装 SDK 或配置 PATH。 Download, unzip, double-click — the compiler and standard library are built in. No SDK install, no PATH setup.

三步开始：① 打开 Zan IDE　② New Project 选模板　③ 按 F5 运行。就这么简单。 Three steps: ① open Zan IDE　② New Project from a template　③ press F5. That's it.

界面布局Layout

  • 菜单栏 / 工具栏：新建、打开、运行、构建、发布等常用操作。Menu & toolbar: new, open, run, build, publish.
  • 项目树：左侧展示当前项目的文件与目录。Project tree: files and folders of the current project.
  • 编辑器标签页：多文件切换，.zform 可切设计/代码视图。Editor tabs: multiple files; .zform toggles Design/Code.
  • 输出面板：编译输出、运行日志、错误列表（点击错误跳转）。Output panel: build output, run logs, error list (click to jump).

目标：单文件 < 20MB、启动 < 500ms、常驻内存 < 100MB。纯软件渲染，只用系统原生 API，不依赖 Electron / Qt / Skia。 Targets: <20MB binary, <500ms startup, <100MB RAM. Pure software rendering on native OS APIs — no Electron / Qt / Skia.

可视化设计器Visual designer

.zform 是可视化界面文件。打开即进入拖拽设计，用工具栏 「设计 / 代码」 在同一标签内切换视图。 A .zform is a visual UI file. Open it to drag-and-drop; the Design / Code button toggles views within the same tab.

  • 拖拽控件到画布，属性面板调样式与数据绑定。Drag widgets onto the canvas; tune style and bindings in the property panel.
  • 「设计 / 代码」只切换当前 .zform 的视图，不会新建文件或标签。Design/Code only toggles the current .zform — it never creates a new file or tab.
  • 非 .zform 文件不能进入设计模式（按钮置灰）。Non-.zform files cannot enter design mode (button disabled).

内存与 asyncMemory & async

值类型在栈上、赋值即拷贝；引用类型（class）在堆上，由 ARC 自动引用计数管理，引用归零即确定性析构。用 weak 打破循环引用，集合支持写时复制（COW）。 Value types live on the stack and copy by value; classes live on the heap, managed by ARC with deterministic destruction at refcount zero. Use weak to break cycles; collections support copy-on-write.

arc.zan
var buf = new Buffer(1024);   // refcount = 1
weak Node? parent;             // 弱引用weak reference

async Task<string> Fetch(string url) {
    var http = new HttpClient();
    var res = await http.Get(url);
    return res.Body;
}

示例：小游戏Example: Mini games

游戏引擎内建游戏循环、Canvas 绘制与输入。继承 Game，实现 Update 与 Draw 即可。附带 Pong、贪吃蛇、打砖块示例。 The engine provides a game loop, Canvas drawing and input. Subclass Game, implement Update and Draw. Ships with Pong, Snake and Breakout samples.

snake.zan
using Zan.Game;

class Snake : Game {
    List<Vec2> body = new List<Vec2>();
    Vec2 dir = new Vec2(1, 0);

    override void Update(float dt) {
        if (Input.Key(Key.Up))    dir = new Vec2(0, -1);
        if (Input.Key(Key.Down))  dir = new Vec2(0,  1);
        Step();
    }

    override void Draw(Canvas c) {
        c.Clear(Color.Black);
        foreach (var seg in body)
            c.FillRect(seg.X*16, seg.Y*16, 15, 15, Color.Green);
    }
}
