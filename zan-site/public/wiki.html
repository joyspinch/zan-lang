安装与构建Install & build

Zan 以源码分发。从仓库构建编译器（需 LLVM 17+ 及开发库）与自带 IDE。 Zan ships as source. Build the compiler (needs LLVM 17+ with dev libraries) and the bundled IDE from the repo.

build
git clone https://github.com/joyspinch/zan-lang.git
cd zan-lang
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
cmake --build .

构建完成后，编译器位于 build/zanc（Windows 为 zanc.exe）。 After building, the compiler is at build/zanc (zanc.exe on Windows).

命令行编译Compile from CLI

zanc
# 单文件single file
./zanc hello.zan --auto-stdlib && ./hello

# 多文件：显式传入全部 .zan 源文件multi-file: pass every .zan source
./zanc src/main.zan src/utils.zan --auto-stdlib

可视化设计器Visual designer

.zform 是一个可视化界面文件。打开它即进入拖拽设计模式，用工具栏的「设计 / 代码」按钮在同一标签内切换视图。 A .zform is a visual UI file. Open it to drag-and-drop; use the Design / Code button to switch views within the same tab.

  • 拖拽控件到画布，属性面板调样式与绑定。Drag widgets onto the canvas; tune style and bindings in the property panel.
  • 「设计 / 代码」只切换当前 .zform 的视图，不会新建文件或标签。Design/Code only toggles the current .zform view — it never creates a new file or tab.
  • 非 .zform 文件不能进入设计模式（按钮置灰）。Non-.zform files cannot enter design mode (the button is disabled).
  • 代码视图里保存写回配对的代码后置文件，不会覆盖表单。Saving in code view writes the paired code-behind, never the form JSON.

调试与发布Debug & publish

调试Debug

在行号槽点击设断点，启动调试后可单步、查看变量、调用栈；热重载仅在运行/调试会话期间生效。 Click the gutter to set breakpoints; step, inspect variables and the call stack during a debug session. Hot reload is active only while running/debugging.

Build → Publish 选择目标平台，IDE 交叉编译出单个原生可执行文件。目标始终是当前项目。 Build → Publish, pick target platforms, and the IDE cross-compiles a single native executable. It always targets the current project.

struct Point {
    public float X;

}

    private T item;
    public Box(T v) { item = v; }
    public T Get() => item;
}

T Max<T>(T a, T b) where T : IComparable<T> {
    return a.CompareTo(b) > 0 ? a : b;
}

内存与 asyncMemory & async

值类型（primitive/struct/enum/tuple）在栈上，赋值即拷贝；引用类型（class）在堆上，由 ARC 自动引用计数管理，引用归零即确定性析构。用 weak 打破循环引用。 Value types (primitive/struct/enum/tuple) live on the stack and copy by value; reference types (class) live on the heap and are managed by ARC — deterministic destruction at refcount zero. Use weak to break cycles.

arc.zan
class Buffer {
    private byte[] data;
    public Buffer(int size) { data = new byte[size]; }
    ~Buffer() { Console.WriteLine("freed"); }
}

var buf = new Buffer(1024);   // refcount = 1
weak Node? parent;             // 弱引用weak ref

async / await

async.zan
async Task<string> Fetch(string url) {

ApiController.zan
[Api(post, route="/api/gather", title="收集数据",
     auth, rank=3, lock="user")]
static void Gather(HttpContext ctx) {
    string name = ctx.InText("name");   // 统一过滤取参filtered input
    ctx.Json("{\"ok\":true}");
}
